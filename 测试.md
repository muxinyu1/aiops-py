# 执行路径追踪测试记录（Java Agent 全栈版）

测试日期：2026-07-01  
测试环境：Linux (Docker + 阿里云 ACR 镜像)  
追踪方案：**Java Agent (Byte Buddy)** — 方法级别全调用栈捕获  
追踪协议：请求头 `X-Return-Trace: true` → 响应体 JSON span 数组（超过 4KB 自动转到 body）

---

## 追踪方案说明

采用自研 Java Agent (`trace-agent-1.0.0.jar`)，基于 Byte Buddy 在类加载时对目标包下所有方法进行插桩。
相比之前仅捕获 Controller 层单一 span 的 AOP 方案，Agent 方案可捕获完整的函数调用栈（含 Service、Repository、Filter 等各层）。

**Agent 配置：**
```
-javaagent:/opt/trace-agent.jar=packages=<target_package>
```

**排除规则：**
- `.tracing.` 包（追踪基础设施自身）
- CGLIB 代理类（`$$` 开头的类）
- getter/setter/toString/hashCode 等
- Spring/Apache/Druid 等框架包

## 关键结论

1. **完整函数调用栈**：Java Agent 成功捕获从 Controller → Service → Repository → Framework 的完整方法级调用链，而非仅 Controller 层单一 span。

2. **父子关系清晰**：每个 span 通过 `parent_span_id` 建立层级关系，可重建完整的调用树。

3. **异常传播可视**：错误从底层（如数据库约束违反、租户验证失败）层层向上传播，每一层都标记 `[ERROR]`，清晰展示异常冒泡路径。

4. **框架内部行为可见**：
   - Apollo: `RepositoryAspect` + CAT 事务追踪 + JPA 生命周期钩子
   - yudao-cloud: 完整 Filter Chain（XSS → Token认证 → 租户验证 → 权限检查 → 异常处理）

5. **性能热点定位**：`duration_ns` 精确到纳秒，可定位热点方法（如 yudao 的 `TenantFrameworkServiceImpl.validTenant()` 占 69ms）。

6. **低侵入性**：仅需在容器启动时加 `-javaagent` JVM 参数 + 挂载 jar 文件，无需修改业务代码。

7. **GET/POST 均支持**：两种 HTTP 方法、正常/异常场景都能正确追踪。
